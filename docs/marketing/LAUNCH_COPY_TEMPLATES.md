## Product Hunt Maker Comment

I built `<PROJECT_NAME>` after repeatedly losing screenshots, downloads and research files across projects. Traditional folders only describe where a file lives, but the same file can relate to a project, person, event and topic.

`<PROJECT_NAME>` creates those relationships locally, uses them for semantic search and project discovery, and can propose safe organization actions. It works without an API key, exposes a read-only MCP server, and every rename or move can be undone.

The first release focuses on macOS and Windows. I would value feedback on search quality, project detection and resource use on 8GB machines.

## 90-second Video Voiceover

My Downloads folder had thousands of files with names like screenshot, final-v2 and document-copy.

Folders only told me where a file lived. They did not tell me what project it belonged to, what it was about, or why I downloaded it.

So I built `<PROJECT_NAME>`.

It scans only the folders I authorize and creates a private context graph on my computer. It connects files to projects, topics and related work using metadata, OCR and local embeddings.

Now I can search for “the Google Play payment error from last month,” even when the filename contains none of those words.

The app explains why files are related. It can preview better names and destinations, but it never lets an AI directly move files. Every action goes through a transaction and can be undone.

It also exposes a read-only MCP server, so AI agents can query the context I approve without receiving unrestricted disk access.

No cloud is required. No API key is required. And it is designed to run on an 8GB computer.

`<PROJECT_NAME>` is open source. Try it, break it, and tell me where the graph is wrong.
