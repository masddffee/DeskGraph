## Current milestones

M2 Content Intelligence remains on the critical path; M3 Hybrid Retrieval, M4 Project Graph, M5 Safe Organization, M6 Watch Mode, M7 Read-only MCP, M8 Product UI, and the first M9a packaged-scope slice are **in progress**. M0 remains open for remote CI evidence, and M1 remains open for complete Windows runtime, peak-memory, and live scan-control evidence. Smart Cleanup Inbox is an explicit v0.1 release requirement. ADR-029 provides a usable, explicitly refreshed, path-free Inbox read model over current exact-duplicate, explicit numeric-version and screenshot-review evidence, with CLI plus four-language Desktop entry points and hard no-action flags. ADR-030 adds an independent immutable Cleanup ActionPlan Preview core with complete target/keeper evidence binding. ADR-031 now exposes a user-invoked local Desktop review boundary: only after an explicit Review action may current paths enter a transient response; the user then chooses the permitted target/keeper pairing and creates a durable path-free, non-executable Preview. Paths are not stored in the plan, history, logs, preferences, or catalog, late asynchronous results are invalidated on scope/refresh/view changes, and no confirmation, Trash, Delete, Execute, Recovery, or Undo entry exists. The combined slices still add no duplicate persistence, background scheduler, action confirmation, reclaimable-space claim, system-trash action, or cleanup Undo. M9a now has a locally verified macOS authorization foundation: the WebView no longer accepts a typed scope path; a Rust-owned native picker creates a versioned security-scoped bookmark, migration 0020 stores the opaque grant separately from general scope responses, startup restores and revalidates it, and every Desktop scan/OCR/search/preview/history/Watch path is gated to the currently active grant. Legacy, corrupt, stale-unrecoverable, cross-platform, or release-invalid grants become `needs_reauthorization`; the debug-only selection receipt is rejected by release builds. The checked-in App Sandbox profile adds only user-selected read/write, app-scoped bookmarks and the outbound-client entitlement required by the current-host WebKit NetworkProcess. A same-bundle ad-hoc A/B proves the UI is blank with a fixed WebKit network-permission exit without that key and renders with the native picker when only that key is added. Production CSP excludes development HTTP/WebSocket origins, `network.server` is absent, core use requires no network, and no remote content/upload client exists. This is still not signed runtime proof: the app lacks an approved product identifier, selected macOS floor, Developer ID signature/notarization, clean-machine no-egress plus arm64/Intel-or-Universal evidence, and the ADR-027 protected-container replacement probe. Windows package identity remains unimplemented. Therefore every production file-action adapter remains fail closed while Windows handle execution and System Trash also remain open. CLI exposes Rename Preview, Status, Smart Cleanup Inbox, and path-free History; Desktop exposes the six-view workspace shell with current backend-backed capabilities plus explicit Cleanup review/Preview creation. The current Desktop shell has dependency-free English, Traditional Chinese, Simplified Chinese, and Japanese catalogs, conservative first-launch locale detection, explicit local preference persistence, and live packaged-app switch/restart evidence. UI locale coverage does not expand the currently verified Traditional Chinese/English extraction, search, or OCR language boundary, and the new Projects, Inbox, History and Settings views remain honest entry surfaces rather than completion of their later full workflows.

Five M2 extraction slices run end to end in core/CLI; the explicit Desktop Screenshot OCR control is contract-, unit-, and build-verified, while live control interaction remains pending. Explicit already-scanned text, Markdown, source-code, text-layer PDF, DOCX, PPTX, XLSX, PNG, JPEG, GIF, WebP, BMP, and TIFF files are resolved inside their authorized scope; the core revalidates canonical scope, exclusion policy, manifest snapshot, and open-file identity; bounded providers receive only a controlled `Read + Seek` or owned validated encoded bytes; durable SQLite jobs support cancellation and interrupted recovery. Text publishes atomically as tagged byte/page/paragraph/slide/cell/OCR-observation provenance plus complete untrusted SQLite/FTS chunks. Images publish signature-checked format and encoded dimensions into a separate structured table, with no pixel decode, EXIF, GPS, filename, or path. The macOS arm64 Apple Vision provider requires `zh-Hant` and `en-US`, keeps normalized top-left boxes and confidence, and has a real local OCR→FTS smoke. Eligible Desktop search results now expose an explicit local-only action whose Tauri commands accept only `scope_id + node_id` or an opaque job ID; durable queued/running/cancelled/completed/error states are visible and no OCR text or path enters the job contract. Windows provider code is package-identity gated, requests `zh-TW`/`en-US`, validates the resolved recognizers as Traditional Chinese/English, stores mandatory boxes with optional confidence, rejects untransformable rotation, and uses terminal-only async close plus bounded single-worker cleanup. Host policy/state-machine tests and Windows cfg check/Clippy pass. D-015 now has a workspace-only, versioned and bounded evaluator plus a macOS Vision runner. The runner binds a private asset manifest/root to exact corpus/image digests, shares production OCR request/output validation, and writes sensitive no-clobber evidence without logging OCR text or paths. A one-case private synthetic mixed-language smoke completed outside the restricted development sandbox with zero CER, three valid observations, and explicit missing RSS/cancellation/packaging/spatial-ground-truth evidence; it is not a representative corpus or release result. M2 is not complete because real Windows/MSIX OCR/cancellation/cleanup, Tesseract and PP-OCRv6 runners/bake-off and fallback implementation, representative corpora, full cross-platform runtime, actual Apple cancellation, live Desktop interaction, and 8 GB extraction benchmarks remain open. Verified Tesseract model hashes do not constitute an accepted runtime; PP-OCRv6 remains an equal candidate rather than an assumed future upgrade.

One M3 lexical vertical slice now runs end to end. Transactional SQLite FTS5 trigram indexes cover current authorized display paths and active extracted chunks; normalized quoted queries are limited to 3–256 Unicode characters, results/candidates are capped, stale content is filtered by source-of-truth joins, and deterministic fusion exposes exact filename/path/content explanations. Scope, metadata-vs-content source, ASCII-alphanumeric extension, and inclusive/exclusive UTC modified-time filters are validated by both retrieval and database boundaries. CLI and Desktop return user-requested paths and bounded snippets while logs omit query/path/text. This is not M3 completion: one- and two-character search, project/folder filters, vector/embedding adapters, semantic and “files like this” queries, hybrid fusion, representative multilingual evaluation, 100k/8 GB/cross-platform benchmarks, and live-UI evidence remain open.

One M7 read-only MCP vertical slice now runs end to end on local macOS arm64. An independently launched local stdio server exposes exactly one `search_files` tool over an existing-schema SQLite read-only/query-only capability. Positive launch grants must reference completed scans and are revalidated on every call; closed schemas accept no paths, there is no writer/scanner/extractor/model/file-action capability, and metadata is the default. Paths and explicit content snippets retain separate untrusted-data labels plus fixed instruction boundaries. Input, request ID, tool content, complete protocol frame, path, snippet and result sizes are bounded; SQLite configures one-second lock wait and two-second progress interruption. Fixed JSON stderr audit fields omit query/path/content/database values, including CLI parse failures and unknown/malformed tool calls. Real child-process tests cover stable MCP lifecycle and negotiation, the exact one-tool registry/annotations, allowed and denied scopes, filename/content injection-as-data, unknown fields/tools, oversized input and oversized echoed-ID recovery, privacy-safe help/version/invalid arguments, missing-database no-create, unchanged main-database/source bytes, and sensitive-log absence. An expired deterministic query proves interruption cleanup and subsequent-request recovery; real wall-clock lock/long-query evidence remains open. SQLite may still create/update adjacent WAL/SHM coordination sidecars for a concurrent reader; on macOS/Linux the exact bundled VFS applies `O_NOFOLLOW` to WAL/SHM and existing sidecars are checked as ordinary non-links before/after open. Windows and other unproven targets deliberately fail closed until equivalent safe VFS/snapshot evidence exists. M7 is not complete: ancestor swap hardening, remaining accepted minimum-necessary read tools, Linux/macOS Intel/client/installer runtime, packaging, broader agent evaluation and release evidence remain open.

Eight M4 vertical slices now run end to end. An explicit already-scanned folder resolves to bounded deterministic facts and explainable Project Suggestions. Stable Project roots and exact-byte duplicate pairs have immutable evidence plus append-only correction. ADR-032 adds bounded manifest-only Project Discovery for one explicitly selected active scanned scope: ordinary results are path-free, current paths appear only in an explicit transient detail, per-scope readiness comes from durable SQLite rather than the bounded recent-job list, and accept/reject repeats current evidence validation without creating membership or file actions. Two different current files can also produce a directional filename-version suggestion only when both pass canonical scope, manifest/platform/open-handle identity, and metadata validation before and after analysis; their UTF-8 names must share an NFC/lowercased base and extension and end in allowlisted numeric `vN` suffixes. The lower number is older, the higher number is newer, evidence is immutable at 9000 basis points, and no content/time/size/model guess is used. Version decisions repeat live verification, bind append-only feedback to equivalent ordered nodes/base/extension/version/provider evidence, make same decisions idempotent, retain opposite corrections, and return changed directional evidence to `suggested`. ADR-028 and migration 0021 add a separate multi-member screenshot-review candidate rather than weakening binary relations: only current same-scope PNG/JPEG/WebP image metadata plus completed active screenshot-OCR provenance can be grouped by exact dimensions and a ten-minute window, at fixed low confidence, without reading OCR text, pixels, filenames, embeddings, or model output. Every immutable member snapshot revalidates current location/file/image/OCR evidence; stale observations fail closed, and path-free history reports currentness. Explicit group discovery/status may return current selected paths. It never claims content similarity, keeper, disposability, reclaimable bytes, or cleanup authorization. No M4 slice performs a file action. This is not M4 completion: file membership, entities/topics, related/similarity/general-version discovery, background Project/duplicate discovery, broader screenshot discovery, cross-pair/root learning, clustering correction, merge/split, retrieval filters, cross-platform runtime, and scale/memory evaluation remain open.

M5 has a fail-closed transaction-protocol foundation, not a working execution vertical slice. A same-parent already-scanned regular file receives an immutable SHA-256/root/parent/source-bound preview and closed append-only journal. Migration 0019 adds immutable request receipts, strictly reduced states, per-plan lease coordination and bounded observation-only recovery records. Deterministic unit fixtures exercise no-overwrite execute/Undo and recovery transitions through a test-only macOS/Linux leaf-name prototype. Independent review and official platform research found that those Unix syscalls cannot atomically bind the source leaf to the exact held inode. A deterministic last-check counterexample now moves a replacement file to the destination while the expected inode remains elsewhere. ADR-026 therefore resolves D-018 by rejecting general Unix Rename/Move execution. A proposed adjacent action lock was also removed because its inode could be replaced by another process. ADR-027 accepts the identity-first topology and Windows primitive contract. M9a now implements only the local macOS native picker/bookmark/active-scope foundation and checked-in entitlements; it does not supply signed package identity, the protected-container fence proof, a platform fence, Windows package identity, or an action adapter. Production builds therefore still return `action_platform_rename_unsupported` before database, journal, lock, or filesystem side effects, and CLI/Desktop expose no Execute, recovery, or Undo control. The Desktop can render all path-free journal states in four languages for durable history compatibility. M5 remains open for signed M9a platform evidence, conditional macOS and Windows fence implementations, Windows handle execution, child-process/fault matrices, Move planning, folders/case-only/cross-volume/System Trash, manifest refresh, and execution UI.

Four bounded M6 vertical slices now run end to end. Untrusted path hints are canonical-scope validated, temporary/hidden entries are ignored, and active events coalesce durably per scope. Exact-audited `notify 8.2.0` feeds macOS FSEvents, Windows `ReadDirectoryChangesW` and Linux inotify into one bounded non-blocking source; callbacks never touch SQLite or the filesystem. Watch-set changes, overflow, rescan, source failure, unmatched routing, a second distinct path, identity replacement, create/delete, directory, hard-link, cross-scope and every ambiguous case request monotonic full-scope recovery. All-scope recovery is recorded in one SQLite immediate transaction, and recovery received during an existing multi-batch scan survives until a fresh follow-up scan. The packaged Desktop coordinator now additionally requires an active platform access grant for both native registration and periodic reconciliation; completed legacy or restore-failed scopes are excluded, while the CLI/core coordinator retains its explicit authorization contract. The fourth slice accelerates only a Unix existing regular-file modify: `/` through root, parent and leaf are opened component-by-component without following links; two anchored snapshots plus exact SQLite root/parent/location/node/topology CAS must agree; one-link/one-present-location and no-active-scan rules must hold. Its atomic delta updates only file/node metadata, invalidates that node's stale content/image outputs, and completes without a scan job; siblings and edges are untouched. Windows and any failed proof remain on full scope. The five-minute missed-event path, bounded coordinator, Tauri scan gate, restart recovery, path-free v2 status, and four-locale Desktop UI remain. This does not complete generic incremental Watch Mode: create/delete/rename/subtree delta, incremental extraction/indexing, cloud-placeholder handling, pause/battery/thermal/low-memory policy, notifications, Smart Inbox, Windows/Linux/macOS Intel runtime/load evidence, tray/autostart, and production-scale multi-language interaction remain open.

## Unresolved blockers

- No GitHub repository/remote and invalid GitHub authentication: remote Issues, Releases, and CI results do not exist.
- Signing, notarization, clean Windows/macOS VM validation, and launch accounts are external later-stage requirements.
- The current 522-package lock was scanned against 1,160 cached RustSec advisories: zero known vulnerabilities, 16 unmaintained warnings, and one `glib` unsound warning. The 17 warnings exactly match the prior 492–518-package locks and trace to existing Tauri/Linux transitive paths; neither target-specific OCR/evaluator/native-Watch/MCP nor the four-package M9a dialog delta adds an advisory. The crates.io cache-lock warning did not prevent the lockfile scan. R-010/R-016 remain open for upstream replacement, notices, SBOM, and release-platform review rather than a missing scan.
- Windows open-handle identity and OCR Rust cfg adapters compile for `x86_64-pc-windows-msvc`; the OCR check/Clippy uses a host-pkg-config bypass only to typecheck Rust. A normal complete scanner/extractor check cannot be produced on this macOS host because bundled SQLite needs a Windows C/MSVC toolchain. No Windows link/runtime is claimed; remote Windows CI remains required.
- Local 10k timing and idempotency are measured, but peak RSS sampling was denied by the restricted runner and its escalation reviewer was unavailable due tool quota. This does not block code work; the 8 GB release gate remains open.

## M5 D-017 system-trash guarantee research checkpoint — 2026-07-19

- macOS candidate — Apple documents `FileManager.trashItem(at:resultingItemURL:)` as moving an item to Trash and returning the actual possibly collision-renamed URL. A security-scoped bookmark created from that URL is the only supported restart receipt candidate. Apple does not document that `trashItem` preserves file-reference identity semantics or atomically binds the call to an already-open source. General `moveItem` rejects file-reference URLs, may copy then remove across volumes, and documents only that an already-existing destination fails; it is not accepted as an identity-bound atomic Undo primitive.
- Windows candidate — Microsoft documents `ITransferSource::RecycleItem` returning the recycled `IShellItem`; an absolute PIDL may be extracted and later reconstructed, and `ITransferSource::MoveItem` with `TSF_NORMAL` documents fail-if-exists behavior. Microsoft does not guarantee that arbitrary Recycle Bin PIDLs remain valid across process, Explorer, session, reboot or OS changes, that the Recycle Bin namespace always binds the transfer handler, or that the source Shell item creates a held-handle-bound mutation. Every receipt failure must therefore be conditional `needs_attention`, never a guessed restore.
- Repository consequence — migration 0022 correctly keeps existing plans immutable and Preview-only, but those rows do not contain the future immutable original-destination and platform-receipt companion needed for restart Undo. They cannot be upgraded or silently confirmed. A later forward migration must apply only to newly created plans and add a closed command/journal/receipt family without widening Rename tables.
- Decision — D-017 remains open and no confirmation or action command is added. The platform candidates may proceed only as disposable signed-runtime probes using synthetic scopes. Acceptance requires exact-source hostile replacement, receipt creation/reopen, same-name Trash collision, original-destination collision, external empty/change, scope revoke, volume detach, process kill at every durable boundary and post-Undo identity/hash verification. Any unproven platform/volume remains Preview-only.
- Research tooling — Firecrawl was discovered and invoked first but returned HTTP 402; official Apple and Microsoft primary documentation was then used through the permitted fallback. The current `objc2-foundation 0.3.2` bindings already expose `NSFileManager` Trash/move APIs behind the existing `NSFileManager` feature, so this research adds no dependency or lockfile change.
